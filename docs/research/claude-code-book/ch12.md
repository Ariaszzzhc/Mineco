# 第 12 章：MCP 集成与外部协议

来源：[MCP 集成与外部协议](https://github.com/lintsinghua/claude-code-book/blob/main/%E7%AC%AC%E4%B8%89%E9%83%A8%E5%88%86-%E9%AB%98%E7%BA%A7%E6%A8%A1%E5%BC%8F%E7%AF%87/12-MCP%E9%9B%86%E6%88%90%E4%B8%8E%E5%A4%96%E9%83%A8%E5%8D%8F%E8%AE%AE.md)

## 本章核心观点

本章把 MCP 视为外部能力协议桥。MCP 的价值在于协议即契约、传输无关和安全边界内嵌。MCP server 提供工具、资源和 prompt，但客户端 runtime 仍负责能力映射、命名、连接管理、权限和审计。

书中还关注多传输协议、配置作用域、连接状态、bridge 双向通信、工具命名和权限模型。

## 值得参考

最值得参考的是“MCP 不是权限来源”。Mineco 应把 MCP tool 映射成普通 ToolDefinition，再走同一套 ToolRuntime、Permission Pipeline、sandbox、audit log。

第二个参考点是工具命名稳定性。MCP server/tool 的命名必须避免与内置工具冲突，并能清楚追踪来源。

## 暂不采纳

Phase 2 不需要实现多传输全部支持。可以从 stdio/local process 起步，HTTP/SSE/WS 作为后续能力。

## 后续建议

Phase 2 文档应补 MCP connection state、tool naming convention、schema discovery strategy、server trust policy。
