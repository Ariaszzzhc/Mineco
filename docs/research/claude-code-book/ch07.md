# 第 07 章：上下文管理

来源：[上下文管理 - Agent 的工作记忆](https://github.com/lintsinghua/claude-code-book/blob/main/%E7%AC%AC%E4%BA%8C%E9%83%A8%E5%88%86-%E6%A0%B8%E5%BF%83%E7%B3%BB%E7%BB%9F%E7%AF%87/07-%E4%B8%8A%E4%B8%8B%E6%96%87%E7%AE%A1%E7%90%86-Agent%E7%9A%84%E5%B7%A5%E4%BD%9C%E8%AE%B0%E5%BF%86.md)

## 本章核心观点

本章把上下文窗口作为预算资源管理。有效窗口不是模型最大窗口，而是扣除输出预留后的可用输入空间。上下文管理需要阈值、渐进式压缩和失败断路器。

四级压缩策略的重点不是名称，而是顺序：先做低成本、低损耗处理，再做高成本摘要。压缩失败不能无限重试。

## 值得参考

最值得参考的是断路器。长任务里上下文压缩失败很容易变成循环性成本浪费。Electrolyte 应在 context manager 中记录连续压缩失败次数，并在达到阈值后进入 terminal 或 degraded mode。

第二个参考点是“压缩不只是摘要”。Snip、artifact、collapse、summary 应按损耗递增排列。

## 暂不采纳

不需要照搬具体阈值数字。阈值应跟 provider/model capability 和实际 token counter 绑定。

## 后续建议

建议补充 ContextBudget 类型，字段包括 modelWindow、reservedOutput、effectiveWindow、warningThreshold、compactThreshold、blockingThreshold、compactionFailures。
