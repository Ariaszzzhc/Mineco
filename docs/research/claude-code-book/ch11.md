# 第 11 章：技能系统与插件架构

来源：[技能系统与插件架构](https://github.com/lintsinghua/claude-code-book/blob/main/%E7%AC%AC%E4%B8%89%E9%83%A8%E5%88%86-%E9%AB%98%E7%BA%A7%E6%A8%A1%E5%BC%8F%E7%AF%87/11-%E6%8A%80%E8%83%BD%E7%B3%BB%E7%BB%9F%E4%B8%8E%E6%8F%92%E4%BB%B6%E6%9E%B6%E6%9E%84.md)

## 本章核心观点

本章把 skill 视为声明式、可复用的任务能力，而插件是可分发的能力包。Skill 的关键在 `SKILL.md` frontmatter、使用场景、参数替换、allowed tools、模型选择、上下文模式和附属文件。

书中还强调技能加载来源、去重、路径条件激活、插件缓存和安全文件提取。

## 对照 Mineco 现有文档

已覆盖：

- [AI Agent Runtime 设计](../../architecture/agent-runtime-design.md) 有 Agent Skills 设计、SkillPackage、SkillActivation、PluginPackage。
- Phase 3 专门规划 Agent Skills。
- Process 文档已有 Agent Skills 规范。

部分覆盖：

- 现有 SkillPackage 偏长期架构，还没有落到 `SKILL.md` 解析规则和参数替换细节。
- allowed-tools 的最小权限原则已有，但可以更具体。
- 插件缓存和技能附属文件的安全提取尚未设计。

## 值得参考

最值得参考的是：Skill 应该是“声明能力需求”，不是直接拿系统权限。Skill 的 allowed tools 应参与正常 ToolRuntime 过滤和权限审批。

`SKILL.md` 作为用户可编辑格式很适合 Mineco，但 frontmatter 字段应从 Phase 3 的最小集开始：name、description、when_to_use、arguments、allowed_tools。

## 暂不采纳

不建议 Phase 3 一开始实现 agent-backed skill、prompt hook skill 或复杂 path activation。先保证手动调用和少量自动激活稳定。

## 后续建议

Phase 3 文档可以补一个最小 `SKILL.md` schema，并规定技能目录读取边界和附属文件访问方式。
