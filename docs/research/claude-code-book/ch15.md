# 第 15 章：构建你自己的 Agent Harness

来源：[构建你自己的 Agent Harness](https://github.com/lintsinghua/claude-code-book/blob/main/%E7%AC%AC%E5%9B%9B%E9%83%A8%E5%88%86-%E5%B7%A5%E7%A8%8B%E5%AE%9E%E8%B7%B5%E7%AF%87/15-%E6%9E%84%E5%BB%BA%E4%BD%A0%E8%87%AA%E5%B7%B1%E7%9A%84Agent-Harness.md)

## 本章核心观点

本章将全书原则收束成自建 Harness 路线：先做对话循环，再做工具系统、权限管线、上下文管理、记忆系统、钩子系统。它强调循环优于递归、schema 驱动、渐进权限、流式优先、可插拔扩展。

它也给出判断：如果任务只是输入到输出转换，不需要 Harness；如果任务需要观察、行动、再观察的循环，并涉及副作用、安全、长会话和恢复，就需要 Harness。

## 值得参考

最值得参考的是实现顺序。Mineco 当前 roadmap 已经符合这个方向：Phase 0 做 loop/tools/approval/store；Phase 1 做可靠性；Phase 2 MCP/tool ecosystem；Phase 3 skills；Phase 4 provider abstraction；Phase 5 长任务。

本章对现有路线的结论更多是“确认方向正确”，不是要求大改。

## 暂不采纳

不采纳书中示例代码层面的接口。Mineco 已经有自己的 ARP 和 Node.js 25-first 技术基线，应保持现有协议源真相。
